Apparently, this is a Windows issue.

Right click your computer icon go to properties and advanced system settings and then under the advanced tab go to system properties. Once there on the advanced tab under performance, click settings then go to the data execution prevention tab and add the game or program to the exception list.

Flavrans November 25, 2018, 4:42am #4

BEX indicates a buffer overflow or DEP exception.

It’s possible that some other software on your computer is interfering with bms.exe, causing it to crash. It can be AntiVirus/Firewall/Security software, game recording software, programs that change Windows font rendering or UI, desktop sharing software, OSD software (for example software that displays gpu stats while playing a game), or malware.

If you are running some AntiVirus/Firewall software, try disabling it temporarily and see if the problem persists. Or, see if you have any other software installed that might interfere with bms.exe and try disabling or uninstalling it.

It can also be a problem with the graphics or audio drivers. See if you can update them to a newer version (if you are not already using the latest drivers for your hardware).

I hope this helps.
