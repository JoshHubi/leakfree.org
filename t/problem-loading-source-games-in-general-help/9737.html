shepters November 25, 2018, 2:43am #17

Am running W7 64 and have been having problems with steam games. As others have said, steam runs, maybe shows an advert and then nothing else happens. This first happened with Portal 2 and now with Black Mesa (BM) but I found a solution.
If you try to run the game exe in the steam folder it reports a bad dll “windows/system32/ avifil32.dl”. Mine is 106KB. My Son has W7 32 90KB and BM works there. So I dropped his dll next to the game exe in my steam folder and it worked.
The idea here is that when an exe loads a dll it looks for it first (if a path is not included in the call) in the exe folder then in the system/registered paths. So the dll is only used for the specific game that needs it and not the whole system.
I have no idea what’s causing this problem but this worked for me. BM exe is called hl2.exe and is located in “Steam folder”\SteamApps"Your Steam ID"\source sdk base 2007.
