Maki November 25, 2018, 4:31am #5

Like Peter said, as far as I know, that’s a generic problem. It happens to me in Half-Life 2 Hammer and Black Mesa Hammer all the time.

The best way I’ve found to overcome it is to simply rock the mouse wheel forward and back (i.e. zoom out and in) which seems to “wake up” the view so the cursor works correctly.

If it’s any consolation, Hammer under Linux (using Wine) has a variation of this problem where the view doesn’t update at all until you hover over it. I mention this because it demonstrates that Hammer does put the other views to “sleep” and is sometimes quite lazy about when to reactivate them, which is probably at the core of this problem.
