Ok, fun as this is, I have something I actually want to show off.

This is a proto program of sorts that I’m planning on turning into something functional and (I think) cool.

455×552 45.3 KB

Basically, it approximates the shape of a fractal by not graphing points in the set, but rather points near the points in the set. I’m sure you all recognize this fractal.

Things start to get interesting when you play with the numbers a bit. Because of the way the program works in approximating the points, with certain input, you can get drastically different graphs every time you run it. Some examples:

459×549 30.9 KB

455×549 33.1 KB

456×550 27.8 KB

457×550 29.7 KB

The reason the Sierpinski gasket stays (nearly indistinguishably) the same every time the program is run is because there are less possibilities with the graphing then when I input different numbers. All four graphs above were created by the same input, but every time the program ran, as you can see, the outcome was drastically different. I can explain this better and in more depth, if any of you are interested, when I make some more headway on the program. Eventually the goal is to release it so anybody can try; I think it’ll be interesting to see what everyone comes up with.
