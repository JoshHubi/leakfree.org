This ‚Äúweak encryption‚Äù with ‚Äúhighly entropic data‚Äù really bugs me. The classic example of shit entropy is ROT-13, where literally the inverse letter of the alphabet is used for both encryption and decryption. However, and this is something we should seriously consider, what if instead of using the same rotational method, you implement a method that changes for every instance. For example, if I have the message:

I can encrypt this using ROT-13 to get:

GURORNEFNAQGURPNGF

Now I‚Äôve got two instances of the exact same grouping of three letters, which most people will narrow down to be ‚ÄúTHE‚Äù. This suggests that this weak encryption method has terrible entropy.

Now my two instances of ‚ÄúTHE‚Äù are masked by the changing rotation, thus resulting in more entropy than before.
