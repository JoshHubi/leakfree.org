Quoting all you so you can see this.

I checked with the programmer in charge of the workshop tool. the addons folder is now no longer supported. You’ll need to be doing development through the workshop. It’s a bit of an annoyance, but think of it this way. By testing your mods through the workshop, you’re testing them as your ‘customers’ would be. Those who play your mod aren’t going to be just downloading and overwriting some files, they’ll just subscribe to the worksop. There are things that change between a local file and the workshop addon, it’s a different environment. By developing through the workshop, you’re able to eliminate those issues completely, as you’ll be closer to the environment that your players will be in. It’s basically and better practice thing.

KevinJRattman August 11, 2020, 11:29am #37
Solareclipsed:

Can someone tell me where this custom folder is located? I also have mods that no longer work, in what folder should I put the mod files now if I want non-uploaded mods to work?

There is a Steam Guide detailing how to set it up (yes, you have to set it up).
https://steamcommunity.com/sharedfiles/filedetails/?id=502670746

You can actually rename the name of the folder as you desire, even to ‘addons’, effectively somewhat “reviving” the old addons folder. Just make sure to change the text accordingly in gameinfo.txt (as shown in the guide, just exchange the word “custom” with “addons”, or any folder name of your choice, preferably without spaces).

Just make sure to always test if your mod works in the Workshop by subscribing it if you are ever going to upload it.
