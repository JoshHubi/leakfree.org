Tei November 25, 2018, 12:32am #8

First of all, use the RUN prompt to start “mplayer2”. if it runs, you have an Explorer bug. if not, then it’s a DLL issue. “mplayer2” is the basic WMP without all the bells and whistles. WMP is built ontop of Explorer. What you have is likely a compatibility issue with WMP and some other Explorer app you have running either intentionally or hidden in the background. if you just want a basic media player, just use WMP Classic. You don’t have to download anything because it’s already on your system. Plus it does everything that WMP does without the headaches and extra BS that WMP does.

You can even set it to run as your default media player.

For a real answer (though using other players is a good idea):

Directly after WMP crashes, go into Control Panel (Start orb > Control Panel), then go into Administrative Tools (if in Category view, change “View by” to “Large icons”).

Go into Event Viewer (might take a few moments to load) and go into Windows Logs. Go through the logs listed inside that folder and find the error that WMP gives, then search the Internets for a solution.

That’s what I’d do.

Then search the Internets for a better player.
