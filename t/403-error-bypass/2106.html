Guf November 24, 2018, 7:04pm #5

If they have secured their borders correctly so that only the proxy machine can get out on 80 then you have no hope of getting around it unless you either have access to the firewall settings or the proxy machine.
If https works it’ll be on 443 and unblockable unless they use a man-in-the-middle https proxy and a custom certificate installation (but that’s pretty obvious). If so, you can get out on 443 to whatever you want (e.g. like a VPN server).
So, in other words, if you can get out on any port, and can set up a server out in the cloud (or connect to one like a VPN), then you can do whatever you want, but if not, then you’re out of luck using their network, unless you stick to their rules.
Note that lots of places forget to block google’s cached pages tho…
