Technology and Gaming Help and Support
Chickenprotector November 25, 2018, 3:45am #1

If people have been getting problems connecting to servers using mods that use the SDK and keep getting “Steam Validation required” as an error when trying to connect to a public server, it’s because Valve screwed up the Source SDK 2006 this afternoon trying to convert it to Steam Pipe.

By some miracle I managed to keep my old GCF files. Some people seem to be getting issues, so I put together a little kit and a readme (simple mind you) do your best to spread the word around if you don’t mind.

This one is temporarily disabled because of the volume of traffic: https://www.dropbox.com/s/gvp1xttqu0i94n6/Keep%20me%21.rar

Updated readme: https://www.dropbox.com/s/cakkzsbq0p4w8bh/Keep%20me!.rar

Download this and disable automatic updates on the tool. (look at the readme) Happy fragging.

YOU DO NOT NEED TO DOWNLOAD THIS IF SERVERS THAT RUN ON THE MOD HAVE BEEN UPGRADED.

Server owners:

From now on, servers need to be installed using SteamCMD with appid 205 (Source Dedicated Server).

That’s about it. The above statement is from another server owner.

Edit: If you can’t access the link do the following steps:
You need to uninstall both SDKs. 2006/2007
Then install 2007 First.
Once you do that, install 2006, let it dawdle.
Then delete the “cache” once it should remain on the list, and your installation should be rolled back.

Alright, like I said, if the link doesn’t work follow these steps:
Go to tools on your Steam library
You need to uninstall both SDKs. 2006/2007
Then install 2007 First.
Once you do that, install 2006, let it dawdle.

MAKE SURE YOU DISABLE AUTOMATIC UPDATES.

Note: Once the servers get updated to the new format, you need to update your client again anyway 'cause you’ll get the same error.
