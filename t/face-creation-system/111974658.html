Yeah, the Face Creation System works using the same flex system used for facial expressions; incidentally, this explains that bug people found where the faces reset on death - flexes are automatically reverted when an npc dies.

I’ve been messing around with the character_manifest.txt, and it looks like npcs that have the same targetname as a set in the list use those settings, otherwise they use a random one for their npc class. You also can set regular expression flexes and make sets for any npc type. For instance, inserting this into character_manifest makes all the vorts bare their teeth at you: npc_alien_slave { flex_data { right_upper_raiser 1.0 left_upper_raiser 1.0 } bodygroup_data { } skin 0 }
