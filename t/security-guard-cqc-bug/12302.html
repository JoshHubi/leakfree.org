Security guard CQC bug

Black Mesa Archive In-game Issues (Mod)
te47 November 25, 2018, 3:26am #1

Just thought I should mention it, in case no one else has… There’s a rather serious bug with the Black Mesa security guards. They’re completely incapable of defending themselves if whatever they’re fighting is sufficiently close to them. This applies to both hostile NPCs and the player (if the player provokes them into attacking).

Basically, if whatever they’re attempting to fight is close enough, the security guards will stop shooting and stand there with their weapons aimed indefinitely. If a zombie corners one, they’ll just stand there and let the zombie continuously swipe at them. If you’re the kind of player that likes screwing with Black Mesa personnel, you can simply run up to security guards to prevent them from being able to fight you. I don’t know if it’d be better to let them shoot at point-blank range or add some kind of melee system like the HECU marines have, but something should really be done about that…

Yeah, I first noticed this bug during the pre-crowbar guard escort sequence. Many times, that guard ended up being cornered by a zombie and became defenseless as a result. Every time that happens, I get a Glock before I get the crowbar, which just seems so wrong…

It’s also a big issue if you like provoking security guards into attacking you. I do that for fun occasionally because I love games that are realistic enough to allow for friendly fire, but I load a save afterwards because I tend to take games seriously. But once you learn that cornering a guard and standing right in front of him makes him shut down, that becomes your default strategy (if damage avoidance is a concern). Which totally kills it.
