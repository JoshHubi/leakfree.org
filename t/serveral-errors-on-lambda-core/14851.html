Flavrans November 25, 2018, 4:52am #3

If the savegame files you are using were saved before June 5, 2016 – 05:53:03 UTC, then they are most likely broken by the v0.3.1 patch, which was released on that date. If this is the case, then you’ll have to start a new game from the chapter you are on.

Savegame files usually only work with a specific version of the game, which means they get broken after an update to the game. Using old savegames can cause a lot of issues in the game, like AI disabled issues, missing/purple textures, visible yellow trigger textures, etc.

They added a savegame file version check in the v0.3.0 update so that you would get an error message if you attempted to load an old savegame. But it seems they didn’t increment the savegame file version number in the v0.3.1 update, which makes it possible to load v0.3.0 savegames in the current version of BM.
