Various game crashes occurs on Dual-Screen Fullscreen setup (Not in windowed!)

Black Mesa Archive In-game Issues (Mod)
Ha3 November 25, 2018, 2:21am #1

Object:

  • Various game crashes occurs on Dual-Screen Fullscreen setup (Not in windowed!)

  • Crash report #1
    Crashing on map load, after defeating the tentacles, going down the ladder to reach the changelevel.
    After loading, the image is frozen, a small sound buffer can be heard, followed by silence, then the same sound buffer can be reheard, then the window closes and hl2.exe has stopped working.

  • Crash report #2
    In some maps, the flashlight is banned otherwise the game crashes (.
    Office complex opening map is the most notable one + some others I don’t remember about.

Hardware:

  • Windows 7 32bit, Radeon X1950 XTX
  • Dual screen multi monitor setup: Primary 1280x1024 on the right 75hz, Secondary 1024x768 on the left 75hz, 100 pixels off the top.
  • Using only Primary as the game monitor.

Normal gameplay performance:

  • Excellent, no noticeable performance loss.
  • The fan runs louder than many other Source games, meaning somehow the game is extremely GPU intensive.

Artifacts:

  • None really

Conclusion:
- Game will crash on Dual-Screen Fullscreen setup.

How to fix:
Either disable Dual-screen in the Windows settings, or, play in Windowed mode, or Noborder Windowed mode:

  • To play in Noborder mode:
  • Open Steam > Library.
  • On tle left, right click on Black Mesa > Properties
  • Click on button Set Launch Options
  • Type this into the field: -windowed -noborder -w 1280 -h 720
  • Replace 1280 with the window width, and 720 with the window height.

Video settings considered as maximum:

  • Model detail: High
  • Texture detail: High
  • Shader detail: High
  • Water detail: Reflect all
  • Shadow detail: High
  • Color C.: Enabled
  • AA: 4x MSAA
  • Filtering mode: Anisotropix 8X
  • Wait for vertical sync: Disabled
  • Motion Blur: Enabled
  • Multicore Rendering: Unselectable (empty box)
  • HDR: Full
