EDIT: I’ve written a small setup tool to automate this process for setting up the 2013 SDK for Black Mesa (you can download the tool from the link in my post on the next page). If it doesn’t work, the original manual setup instructions are below.

Okay so after fiddling around with the Source file directory system, I came up with my own way of manually setting up the Source 2013 compile tools for Black Mesa. I’ve given these instructions to everyone on the Hazard Course team, including some of the testers for their own personal use, and they have it running without any major issues.

That said, here’s how to do it:

  1. Go to the Steam Tools and download Source SDK Base 2013 Singleplayer (Note that Source SDK Base 2013 Multiplayer also works, but I’m going to assume you have the Singleplayer version for consistency).

  2. Go to your [b]…\Steam\SteamApps\common\Source SDK Base 2013 Singleplayer[/b] directory and create a new folder called “BMS”.

  3. Download and SAVE this file (gameinfo.txt) into the BMS directory that you just created.

  4. Open up the gameinfo.txt file that you just downloaded. On Line 17, there is an entry that says:

Game "C:\Program Files (x86)\Steam\steamapps\sourcemods\BMS"

If this is not the path to your installation of Black Mesa, change it so that it is.

  1. Go to [b]…\Steam\SteamApps\common\Source SDK Base 2013 Singleplayer\bin[/b] and launch hammer.exe. If it asks you to choose a configuration, choose any one since it doesn’t matter for now.

  2. In Hammer, go to Tools=>Options…. Click Edit. This will open the Edit Game Configurations window. Click Add, and type “BMS” into the text box, then click OK. Close the Edit Game Configurations box.

  3. Use the drop-down menu to switch to the “BMS” configuration that you just created. It should be mostly blank.

  4. Next to the Game Data files section, click Add. Navigate to your [b]…\Steam\SteamApps\sourcemods\BMS[/b] folder and choose the file “base.fgd”. Repeat this step for the files “halflife2.fgd” and “bms.fgd” IN YOUR BMS INSTALLATION FOLDER. Make sure that you do NOT use the FGD files that are located in [b]…\Steam\SteamApps\common\Source SDK Base 2013 Singleplayer\bin[/b].

  5. You can leave Game Executable Directory blank.

  6. For Game Directory, choose the [b]…\Steam\SteamApps\common\Source SDK Base 2013 Singleplayer\BMS[/b] folder that you created earlier.

  7. For Hammer VMF Directory, choose any folder you want. This is where your raw VMFs will be saved.

  8. Download and SAVE this file (rungame.bat) into a safe folder. MAKE SURE that this folder is AWAY from ALL of your Steam folders!

  9. Open up and EDIT the rungame.bat file that you just downloaded. The first two lines look like this:

SET STEAMEXE="C:\Program Files (x86)\Steam\steam.exe"
SET BMSDIR="C:\Program Files (x86)\Steam\SteamApps\sourcemods\BMS"

Verify that these are the correct paths to your Steam executable and your Black Mesa installation folder, respectively. If they are not, change them so that they are.

  1. Go back to Hammer, and now switch to the Build Programs tab. For the configuration, make sure that the “BMS” configuration that you created is selected.

  2. For Game executable, TYPE (don’t click on “Browse”) the path to wherever you saved your rungame.bat file. For example, my entry says “C:\Users\Ryan\Dropbox\Maps\BMS\rungame.bat”.

  3. For BSP executable, go to [b]…\Steam\SteamApps\common\Source SDK Base 2013 Singleplayer\bin[/b] and choose vbsp.exe.

  4. For VIS executable, go to [b]…\Steam\SteamApps\common\Source SDK Base 2013 Singleplayer\bin[/b] and choose vvis.exe.

  5. For RAD executable, go to [b]…\Steam\SteamApps\common\Source SDK Base 2013 Singleplayer\bin[/b] and choose vrad.exe.

  6. For the Place compiled maps entry, choose the …\Steam\steamapps\sourcemods\BMS\maps folder on your system.

  7. Finally, exit Hammer and restart with the BMS config that you just created. This should now compile your maps properly, AND it should be able to mount ALL of the correct BMS content without weird issues, AND it should be able to run Black Mesa directly, unlike other solutions that use the 2013 SDK.
